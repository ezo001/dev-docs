# Managing DOCX and MD Asset Folders

This project uses a **discovery-based** layout so you can add or retire asset folders without editing config files.

---

## Folder layout

```
my-docs/
├── DOCX/                    ← Source Word files (one subfolder per asset)
│   ├── Thread/              ← Digital Thread .docx files
│   ├── IAI/                 ← Industrial AI .docx files
│   └── AOT/                 ← (example) add when needed
└── MD/                      ← Converted markdown (one subfolder per asset)
    ├── Thread/
    ├── IAI/
    └── AOT/
```

**Rule:** For every asset (Thread, IAI, AOT, etc.), use the **same name** under both `DOCX` and `MD`.

---

## Adding a new asset (e.g. AOT)

1. Create the folders:
   - `DOCX/AOT/`
   - `MD/AOT/` (optional; the converter will create it)
2. Put your .docx files in `DOCX/AOT/`.
3. Run the converter (see below). No config or script edit required.

---

## Converting

**Convert all assets** (every subfolder under DOCX that has .docx files):

```bash
python convert_docx_to_markdown.py --all
```

Uses `DOCX` and `MD` in the current directory by default. From another location:

```bash
python convert_docx_to_markdown.py --all --docx-base "C:\path\to\my-docs\DOCX" --md-base "C:\path\to\my-docs\MD"
```

**Convert one asset** (e.g. IAI only):

```bash
python convert_docx_to_markdown.py DOCX\IAI MD\IAI
```

**Keep existing .md and only add/update** (no clear before convert):

```bash
python convert_docx_to_markdown.py --all --no-clear
```

---

## Sunsetting an asset

1. Remove or archive the .docx from `DOCX/<AssetName>/` (e.g. delete or move `DOCX/AOT/`).
2. Optionally delete `MD/<AssetName>/` if you no longer need that markdown.
3. No config to edit. The next `--all` run will simply skip that folder (no .docx there).

---

## How discovery works

- The script looks at **subfolders of DOCX** only (one level: `DOCX/Thread`, `DOCX/IAI`, etc.).
- A subfolder is converted only if it contains **at least one .docx** (excluding `~$` temp files).
- Output goes to **MD/<same subfolder name>**.
- Subfolders come and go; the script never uses a fixed list of asset names.

---

## Pushing to the site

The site reads directly from **MD/**: Industrial AI from `MD/IAI`, Digital Thread from `MD/Thread`, and any new section from `MD/<folder>`. After conversion, there is **no copy step**—just run `npm start` or deploy.

---

## Will a new folder get a landing page automatically?

**No.** Adding a new folder (e.g. `DOCX/AOT`) and running the converter only creates markdown in `MD/AOT`. It does **not** create a new section or landing page on the Docusaurus site.

You can **automate** the setup with the included script (or batch file).

### Option A: Automated (recommended)

From the project root:

**Windows (batch):**
```batch
add_docs_section.bat aot "AOT Foundation" AOT
```
Third argument is the MD folder name (optional). Use it when your DOCX folder is `DOCX/AOT` so the plugin path is `MD/AOT`.

**Or run the Python script directly:**
```bash
python add_docs_section.py aot "AOT Foundation" --md-folder AOT   # path = MD/AOT (matches DOCX/AOT → MD/AOT)
python add_docs_section.py aot "AOT Foundation"                   # path = MD/aot by default
python add_docs_section.py my-asset "My Asset"                    # path = MD/my-asset
```

The script:

1. Creates **MD/&lt;folder&gt;/intro.md** (landing page with `slug: /`, Accenture styling)
2. Creates **sidebars-&lt;id&gt;.js**
3. Updates **docusaurus.config.js** (plugin **path: MD/&lt;folder&gt;** and navbar item)

Then put .docx in `DOCX/<folder>/`, run the converter, and run `npm start`. No copy step—the site reads from `MD/<folder>`.

---

### Option B: Manual steps

If you prefer to edit by hand:

1. **Add a docs plugin** in `docusaurus.config.js` with **path: 'MD/&lt;folder&gt;'** (e.g. `MD/AOT`), plus `id`, `routeBasePath`, `sidebarPath`.
2. **Add a navbar item** under `themeConfig.navbar.items` (e.g. `{to: '/aot/', label: 'AOT', position: 'left'}`).
3. **Create** `MD/<folder>/intro.md` with `slug: /` and `sidebar_position: 1` (use the Accenture landing layout if desired).
4. **Create** `sidebars-<id>.js` with `{type: 'autogenerated', dirName: '.'}`.
5. Convert .docx into `MD/<folder>/`; the site reads from there.
