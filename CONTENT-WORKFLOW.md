# Content workflow: updating the docs site

## Is there an administrator view?

Docusaurus is a **static site generator**. It has no built-in admin panel or CMS. Content is the markdown/MDX files in the repo; you change the site by changing those files and redeploying.

**Ways to “admin” your content:**

| Approach | Best for | Effort |
|----------|----------|--------|
| **GitHub as admin** | You (or a small team) editing in the browser | None – use “Edit this page” |
| **Local/IDE** | Bulk changes, renames, new folders | Use Cursor/VS Code and push to GitHub |
| **Headless CMS** | Non-technical editors, many pages | Add Decap CMS or similar |

**Recommended:** Use **GitHub as your admin**: each page has an **Edit this page** link (now configured) that opens the file on GitHub. Edit, commit to `main`, and Azure redeploys automatically. No extra tools.

---

## Best practice: making changes

1. **Edit content**  
   - In the browser: open the doc → click **Edit this page** → edit on GitHub → commit to `main`.  
   - Or in your IDE: edit under `docs/`, `docs-industrial-ai/`, or `docs-digital-thread/`, then commit and push to `main`.

2. **Deploy**  
   Pushes to `main` on `eolson1967/docusaurus-foundations` trigger the Azure workflow; the site updates in a few minutes.

3. **Preview before merge**  
   Open a pull request; Azure creates a preview URL so you can check changes before merging.

## Adding Digital Thread files

- **Location:** `docs-digital-thread/`
- **Sidebar:** `sidebars-digital-thread.js` uses `autogenerated`, so any `.md`/`.mdx` you add under `docs-digital-thread/` will appear in the Digital Thread sidebar automatically.
- **Steps:**
  1. Add new files in `docs-digital-thread/`, e.g. `docs-digital-thread/overview.md`, `docs-digital-thread/architecture.md`.
  2. Use front matter for order and titles, e.g.  
     `sidebar_position: 2`  
     `title: Overview`
  3. Commit and push to `main` (or use “Edit this page” on the intro and add links to the new docs).

The existing `docs-digital-thread/intro.md` has `slug: /`, so it stays the landing page for `/digital-thread/`.

## Replacing AOT files with Industrial AI files

Right now **Industrial AI** is served from `docs-industrial-ai/` and those files are AOT-focused. To “replace AOT with Industrial AI” you have two patterns:

### Option A: Replace content in place (same URLs, new content)

- Keep the same file names and paths under `docs-industrial-ai/`.
- Replace the contents of each file with the new Industrial AI content (or overwrite files with your new versions).
- Sidebar and URLs stay the same; only the text changes.
- **Steps:** Copy your new Industrial AI markdown over the existing files in `docs-industrial-ai/`, commit, push.

### Option B: New set of files (new structure)

- Remove or archive the current AOT docs and add a new folder structure for Industrial AI.
- **Steps:**
  1. Back up or delete the current files in `docs-industrial-ai/` (e.g. move them to a backup branch or folder).
  2. Add your new Industrial AI markdown files under `docs-industrial-ai/` (e.g. `intro.md` with `slug: /`, then `getting-started.md`, `architecture.md`, etc.).
  3. `sidebars-industrial-ai.js` is `autogenerated`, so the new files will appear in the sidebar; use `sidebar_position` in front matter to order them.
  4. Commit and push to `main`.

**Recommendation:** If the new Industrial AI set has different doc names/topics, use **Option B**. If it’s the same list of topics with updated text, use **Option A**.

## Quick reference: folder → site area

| Folder | Site section | Sidebar config |
|--------|--------------|----------------|
| `docs/` | Landing (`/`) | `sidebars.js` |
| `docs-industrial-ai/` | `/industrial-ai/` | `sidebars-industrial-ai.js` (autogenerated) |
| `docs-digital-thread/` | `/digital-thread/` | `sidebars-digital-thread.js` (autogenerated) |

Editing these folders (on GitHub or in your IDE) and pushing to `main` is the way you “admin” the site. No separate admin view is required.
