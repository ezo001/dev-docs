# Introduction

Accenture Operations Twin (AOT) is a collection of software accelerators and tools, including Intelligent Advisor, which can be assembled to deliver client solutions. AOT accelerates the integration of product, process, and live data from disparate IT and OT systems, creating a comprehensive and contextualized view of operations to enable better decisions and optimized processes.

Intelligent Advisor (IA) is an AOT component and an AI-based solution that enables distinct types of workers to focus on critical issues with real-time-generated, prioritized, and contextualized insights and recommendations. It combines the following functional components: Insight generation engine, collaboration via MS Teams, Actions, Advisor panel, Insight Viewer template, and Insight lifecycle management.

The user interface for IA is made up of two distinct parts -- the IA side panel and the Template Viewer.

##  Target Audience

End users of the AOT UI

## Purpose

This guide explains how to use AOT\'s Intelligent Advisor UI with Template Viewer.

# Glossary


| > **Term** | > **Definition** |
| --- | --- |
| > AOT (Accenture Operations Twin) | > A suite of software accelerators and tools that integrate product, process, and live data from IT and OT systems, providing a contextualized view of operations for improved decision-making and process optimization. |
| > Intelligent Advisor (IA) | > An AI-based component of AOT that generates prioritized, contextual insights and recommendations, enabling users to focus on critical issues and collaborate via MS Teams. |
| > Insight | > An event or alert generated by business logic, contextualized by user role, providing understanding of operational events. Insights support root-cause analysis, action creation, and team collaboration. |
| > Insight Card | > A UI element displaying details of a single insight, including priority, title, creation timestamp, role, status, description, actions, and ID. |

| > Action | > A task or work activity assigned to address an issue highlighted by an insight. Actions can be manually created or, in future releases, automatically generated by the recommendation engine. |
| > Action Card | > A UI element showing details of an action, such as title, creation timestamp, role, status, due date, action count, work order status, and ID. |
| > Action Status | > New, In Progress, Closed. Describes the lifecycle stage of an action. |

| > Related Actions | > List of actions linked to a specific insight, shown in the Template Viewer. |
| > KPI Breach | > A type of insight indicating that a Key Performance Indicator (KPI) has exceeded its threshold, with drill-down options for further analysis. |
| > Asset Optimization | > A type of insight focused on predictive failure analysis for equipment, visualized with historical probability graphs. |
| > Filtering | > Functionality for narrowing down insights or actions by criteria such as count, sort order, source, status, priority, sub-type, and more. |
| > Collaborate | > Feature allowing users to work together on resolving insights or actions, accessible via dedicated buttons in the UI. |
| > Create Insight / Create Action | > UI options for manually adding new insights or actions, requiring completion of mandatory fields. |
| > Edit Insight / Edit Action | > Pop-up windows for modifying existing insights or actions, with editable fields depending on how the item was generated. |
| > Closed Insights / Actions | > Items that have been resolved and are no longer active, viewable via filter options. |
| > Archived Insights | > Insights automatically closed due to expiration, with system-generated reasoning. |

## Insights

An Insight is an event/alert generated through certain business logic, contextualized by role -- that helps AOT users gain a relevant understanding of that event, with the ability to drill down into the root-cause analysis of that event, create an action against that Insight, or collaborate with team members to resolve the issue.

## Actions

An action is a work activity or a task that is being attributed to a certain person to address the specific issue derived from an Insight.

#### Create New Action

Actions are created manually against the insights. Creating actions manually enables AOT users to resolve the issues highlighted through the insights. In the future, some of the Actions are planned to be generated automatically through our recommendation engine. There are two kinds of Actions. An Action that has a Work Order and an Action without one. The **Create New Action** pop-up screen can be accessed using buttons in either the [Insight Summary](#insight-summary) or the Recommended Action items. To create an Action, the user must complete the fields in the \"Action Details\" tab and then click the \"Create Action\" button.

1.  **Title** -- free-text field of ten to fifty alphanumeric characters

2.  **Plant** -- selection drop-down used to choose a plant.

3.  **Department** -- selection drop-down used to choose a responsible team.

4.  **Assigned to** -- selection drop-down used to choose which role in the previously selected department will have visibility into these Actions in the Action panel.

5.  **Asset Hierarchy** -- selection drop-down used to choose the asset to be linked to the action.

6.  **Topic** -- selection drop-down used to choose the subject.

7.  **Status** -- read-only status of New

8.  **Action by** -- selection calendar used to select a future due date for the Action.

9.  **Action Details** -- free text field of 10-500 alphanumeric characters.

10. **Create Action** -- Submission button enabled once the required fields are filled.

![Create New Action screen](media/image32.png)\{width="7.071201881014873in" height="4.760416666666667in"\}

## Recommendations

Recommendations represent a set of proposed Actions that can be selected to resolve the related insight. These are generated automatically by our recommendation engine by leveraging knowledge from previous Insight-to-action processes, using ML algorithms, and complex rule-based models. This section contains a list of recommended actions. The Action link is currently a placeholder for a future feature.

## More Details

The content in this section varies by Insight category. If the Insights are KPI-based, this section will have KPI tiles and KPI drill-down possibilities. If the Insights are based on Asset Optimization, then it will show the chart with the historical probability of failure of that equipment. Examples of both scenarios are shown below.

**KPI Breach** -- This view shows the following tiles:

-   KPI Tile -- clicking the tile redirects to the KPI drill-down page of the Smart KPI Application

-   KPI Description -- Overview and formula

-   Trendline Timeline

> ![image showing the KPI tiles and drilldown details for insight category](media/image45.png)\{width="10.342856517935259in" height="5.1283344269466316in"\}
