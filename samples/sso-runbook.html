SSO integration runbook

Operational runbook · Case study · Sanitized for public portfolio

Problem

Teams provisioning Single Sign-On for a static documentation site faced scattered Confluence pages, incorrect SAML callback URLs, and repeated stage/prod mistakes. Identity engineers and doc platform owners used different vocabulary for the same configuration surfaces.

My role

I consolidated tribal knowledge into a one-page runbook with explicit stage gates, checklists, and cross-links to a decision flowchart for SAML vs. OpenID Connect paths.

Design choices

Choice Rationale
Task-based structure (top to bottom) Operators execute in order; no narrative to skim past
Three config surfaces Entra app registration, Azure SWA settings, repo config file — matches how teams actually work
Fill-once identity table Reduces copy-paste errors for hostnames and AIR IDs
Stage before Prod gate Prevents prod secrets on stage hosts (common failure mode)
Warning callout at top SAML Reply URL mismatch is a hard blocker — surfaced before any steps

Writing excerpt (sanitized)

# Identity provider → Azure Static Web Apps SSO — one-page runbook

Fill in the blanks, then work top to bottom. Stage must be complete
before Copy to Prod.

## Identity & hosts (fill once)

| Item                  | Your value        |
|-----------------------|-------------------|
| Application record ID | ________________  |
| SWA hostname (Stage)  | ________________  |
| SWA hostname (Prod)   | ________________  |

Redirect URI pattern (Entra on Static Web Apps):
https://<hostname>/.auth/login/aad/callback

## Config surface 1 — Entra app registration
- [ ] Application (client) ID recorded (Stage / Prod)
- [ ] Redirect URI added for Stage host
- [ ] Redirect URI updated for Prod after copy

## Config surface 2 — Azure Static Web Apps settings
- [ ] AZURE_CLIENT_ID matches app registration
- [ ] Secrets stored in approved locations only
- [ ] Authentication mode matches roles design

## Config surface 3 — Repo staticwebapp.config.json
- [ ] Tenant ID placeholder replaced
- [ ] Routes still correct for all doc plugins
- [ ] Push triggers pipeline after Stage secrets are valid

Outcomes

  • Single source of truth replaced 4+ fragmented wiki pages
  • Checklist format usable in change tickets and CAB reviews
  • Linked flowchart doc handles SAML/OIDC branching without bloating the runbook

Lessons

Runbooks fail when they try to teach and execute. This doc executes. Conceptual explanation lives in a separate linked guide — a pattern I reuse for any high-stakes ops doc.
