## Introduction

Industry X Digital Thread Foundations deliver industry-agnostic building blocks that create a virtualization layer over connected products, operations, and services. It provides a communication framework connecting different systems involved in the product lifecycle so that data can be shared, thus leading to improvements in industrial and manufacturing processes. IXDT goes beyond just the data. It provides end-to-end information that is relevant to an asset and thus drives efficiency and value for a business.

### Purpose

This document serves as an index of release notes documents. The releases are sorted by date with the most recent release at the top of the table.

# Release Notes Index


| **Release** | **Notable Changes** |
| --- | --- |
| 1.2 | The 1.2 release introduces flexible, industry-agnostic building blocks that create a virtualization layer across connected products, operations, and services. It features improved Bill of Material (BOM) management through a structured and efficient workflow for converting Engineering BOMs (EBOMs) into Manufacturing BOMs (MBOMs) in batch. The conversion ensures accurate transformation of engineering data into manufacturable formats, with full traceability and version control. The comparison feature provides precise data analysis across systems, highlights discrepancies with full traceability. The new Data Quality Check feature allows users to define quality checkpoints based on their preferences to ensure the accuracy and reliability of data throughout the digital thread. Additionally, the new Digital Thread Dashboard offers comprehensive management access and insights into the entire digital thread ecosystem.. For more details, see the related [Release Notes](https://ts.accenture.com/:b:/r/sites/GlobalDocTemplates/Published%20Documents/IX%20Thread/1.2/DT_Release_Notes_1_2.pdf) document. |
| 1.1 | The 1.1 release of Industry X Digital Thread introduces flexible, industry-agnostic building blocks that create a virtualization layer across connected products, operations, and services. It adds improved Bill of Material (BOM) management through a structured and efficient workflow for converting and comparing Engineering BOMs (EBOMs) into Manufacturing BOMs (MBOMs). The conversion feature ensures accurate transformation of engineering data into manufacturable formats, with full traceability and version control. The comparison feature provides accurate data analysis across systems, highlighting discrepancies and matches full traceability. The capability to generate MBOMs tailored to client needs based on the custom Rules and templates is also new for this release. Alongside these use case enhancements, this release also delivers significant new functionality to the core IX Digital Thread foundational layer. For more details, see the related [Release Notes](https://ts.accenture.com/:b:/r/sites/GlobalDocTemplates/Published%20Documents/IX%20Thread/IXDT%201.1/DT_Release_Notes_1_1.pdf) document. |
| GA | The General Availability (GA) release focuses on two key use cases -- Software-Defined Battery Management System and Business Policy Management. Additionally, the custom UI for the data catalog is now equipped with a search-based interface that enables advanced and efficient data discovery and search. The query engine is now a distributed query engine that can be used to query multiple source systems and is also equipped to create data products as per client needs. Furthermore, several enhancements to the set of connectors and data pipelines that IX Digital Thread utilizes to seamlessly integrate with various source systems along with enhancements to the workflow engine and the monitoring dashboard. For more details, see the related [Release Notes](https://ts.accenture.com/sites/GlobalDocTemplates/Published%20Documents/IX%20Thread/GA/DT_Release_Notes_GA.pdf) document. |
| RC1 | The Release Candidate Once (RC1) release delivers implementation and integration of the tools for Change Data Capture and Query Engine. This release also focuses on new connectors such as IIoT, Jira, and an enhanced SDK version. The Teamcenter and SAP connectors have been enhanced for increased efficiency. Additionally, a custom UI has been created for the data catalog tool. For more details, see the related [Release Notes](https://ts.accenture.com/:b:/r/sites/GlobalDocTemplates/Published%20Documents/IX%20Thread/RC1/DT_Release_Notes_RC1.pdf) document. |
| MVP | The Minimum Viable Product (MVP) release delivers more developed proof of concepts along with fundamental capabilities that enable connected data flows across industrial assets. On top of that, Digital Thread's development, testing, and production workflows are now more efficient due to new application enhancements and automated deployment processes. Together, these improvements establish the foundation for an integrated thread solution that offers enhanced efficiency, insights, and automation. Next, an end-to-end part classification use case highlights Digital Thread's ability to ingest parts data from Teamcenter, enhance information using artificial intelligence and machine learning services, and generate a complete product data record that is used to update the source. Finally, a fully functional demo shows exactly how DT's new dashboard-style UI can be used to manage the part classification process. For more details, see the related [Release Notes](https://ts.accenture.com/:b:/r/sites/GlobalDocTemplates/Published%20Documents/IX%20Thread/MVP/DT_Release_Notes_MVP.pdf) document. |
