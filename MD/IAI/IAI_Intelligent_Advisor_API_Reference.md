## Introduction

Industrial AI Foundation (IAI) is a set of software tools, including Smart KPIs, designed to unify data from various IT and OT systems, giving users a comprehensive view to improve decision-making and processes. Intelligent Advisor (IA), an AI-powered IAI module, delivers real-time insights and recommendations through components like an insight engine, collaboration tools, and lifecycle management, helping users address critical issues efficiently.

### Purpose

This reference document includes information about paths, inputs, outputs, and error management for APIs related to IAI Intelligent Advisor. The document includes descriptions of APIs used for configuration as well as descriptions of APIs that provide middleware functionality.

### Target Audience

Client Delivery Teams planning to deliver Industrial AI Foundation.

An API testing tool such as [Postman](https://app.getpostman.com/app/download/win64).

## Glossary


| &gt; **Term** | &gt; **Definition** |
| --- | --- |
| &gt; IAI (Industrial AI Foundation) | &gt; A suite of software accelerators and tools integrating product, process, and live data from IT and OT systems, providing a comprehensive view of operations for better decision-making and process optimization. |
| &gt; IA (Intelligent Advisor) | &gt; An AI-based IAI component generating real-time, prioritized, and contextualized insights and recommendations, with modules for insight generation, collaboration, actions, advisor panel, and lifecycle management. |
| &gt; API (Application Programming Interface) | &gt; Defined methods and endpoints allowing software components to communicate; used here for configuration, event data retrieval, and machine learning model management. |
| &gt; Asset Events Data | &gt; Data related to specific events (Insights, Actions, WorkOrders) associated with an asset, retrieved using the GetAssetEventsData API. |
| &gt; Insight | &gt; A data-driven observation or recommendation generated by Intelligent Advisor, prioritized and contextualized for operational decision-making. |
| &gt; Action | &gt; A recommended or required operational step, often generated from an insight and tracked within the system. |
| &gt; WorkOrder | &gt; A formal task or job assigned within operations, tracked as an event type in the system. |
| &gt; Configuration APIs | &gt; APIs for creating, updating, or deleting configuration settings for models and insights within the IAI platform. |
| &gt; Entity Viewer | &gt; A micro front end application within IAI for visualizing 360-degree information related to nodes in the asset hierarchy. |
| &gt; Event Type | &gt; A classification for events associated with assets, such as Insight, Action, or WorkOrder. |
| &gt; Content-Type | &gt; HTTP header specifying the media type of the resource or data being sent (e.g., application/json). |
| &gt; Error Management | &gt; Standardized handling of API errors, including HTTP codes (e.g., 400 for Bad Request, 401 for Unauthorized, 500 for Invalid Data). |
| &gt; M/O (Mandatory/Optional) | &gt; M designates that a value is Mandatory, while O designates it as Optional. |
## Events Data APIs

The APIs now integrated into the Intelligent Advisor microservice were originally designed for the Operations Hierarchy component.

The Operations Hierarchy Entity Viewer is a micro front end within the IAI application, created to provide 360-degree visibility of nodes in the asset hierarchy.


| **API** | **Description** |
| --- | --- |
| GetAssetEventsData | Use this API to fetch asset specific events data. Events can be Insights, Actions or WorkOrder. Based on asset external ID, type of event and dimension(event properties), the API will return the event data. |
| GetEventsParameterCount | Use this API to fetch count of asset specific events data types. Based on asset external ID, type of event and event property, the API will return the count of events data. |
