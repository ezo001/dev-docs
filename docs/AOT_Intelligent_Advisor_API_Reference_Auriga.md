# Introduction

Accenture Operations Twin (AOT) is a set of software tools, including Smart KPIs, designed to unify data from various IT and OT systems, giving users a comprehensive view to improve decision-making and processes. Intelligent Advisor (IA), an AI-powered AOT module, delivers real-time insights and recommendations through components like an insight engine, collaboration tools, and lifecycle management, helping users address critical issues efficiently.

## Purpose

This reference document includes information about paths, inputs, outputs, and error management for APIs related to AOT Intelligent Advisor. The document includes descriptions of APIs used for configuration as well as descriptions of APIs that provide middleware functionality.

## Target Audience

Client Delivery Teams planning to deliver Accenture Operations Twin.

## Prerequisites

An API testing tool such as [Postman](https://app.getpostman.com/app/download/win64).

# Events Data APIs

The APIs now integrated into the Intelligent Advisor microservice were originally designed for the Operations Hierarchy component.

The Operations Hierarchy Entity Viewer is a micro front end within the AOT application, created to provide 360-degree visibility of nodes in the asset hierarchy.

| **API** | **Description** |
|:---|:---|
| GetAssetEventsData | Use this API to fetch asset specific events data. Events can be Insights, Actions or WorkOrder. Based on asset external ID, type of event and dimension(event properties), the API will return the event data. |
| GetEventsParameterCount | Use this API to fetch count of asset specific events data types. Based on asset external ID, type of event and event property, the API will return the count of events data. |
